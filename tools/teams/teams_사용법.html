Teams 채팅 읽기 / 쓰기 — 다른 PC로 옮겨 쓰기

특정 Teams 채팅방에 대해 메시지 읽기·보내기 기능만 Windows 11 등 다른 컴퓨터에서 사용하기 위한 최소 구성 가이드.

① 의존성 ② 들고 갈 파일 ③ 사전 설치 ④ 폴더 구성 ⑤ 사용 순서 ⑥ 주의사항 ⑦ 트러블슈팅

① 의존성

외부 의존성은 단 하나입니다.

selenium>=4

나머지는 모두 Python 표준 라이브러리(argparse, json, os, re, shutil, sys, time, urllib, pathlib, typing)만 사용합니다. .env 파서도 스크립트 내부에 자체 구현되어 있어 python-dotenv도 필요하지 않습니다.

Selenium Manager 내장

Selenium 4부터는 ChromeDriver / EdgeDriver를 자동 탐색·다운로드하므로, 드라이버를 별도로 설치하거나 PATH에 등록할 필요가 없습니다.

② 들고 갈 파일

파일필수용도
tools/teams/teams_chat_rw.py 필수 본체 스크립트. 단일 파일(약 1,666줄)로 login, read, write, files 4개 서브커맨드 제공.
tools/teams/teams_requirements.txt 필수 selenium>=4 한 줄. pip install -r로 설치.
.env (직접 작성) 필수 TEAMS_CHAT_URL='...' 한 줄. 또는 매 실행 시 --chat-url 인자로 대체 가능.
tools/teams/README.md 선택 영문 빠른 시작 가이드.
tools/teams/Teams_chat_read_write_manual_cyh.md 선택 한글 상세 매뉴얼·내부 구조 설명.
tools/teams/post_ics_to_teams.py 제외 Workflow Webhook 송신용 — 본 용도에 불필요.
tools/teams/open_ics_in_teams_chat.py 제외 딥링크 기반 송신 헬퍼 — 본 용도에 불필요.

③ Windows 11 사전 설치

Python 3.10 이상

공식 인스톨러 또는 winget:

winget install Python.Python.3.12

설치 시 "Add Python to PATH" 체크 권장.

Chrome 또는 Edge

Edge는 Windows 11에 기본 탑재.
Chrome 사용 시: winget install Google.Chrome

스크립트는 두 브라우저 모두 자동 인식 (--browser auto).

④ 다른 PC 폴더 구성 예시

C:\Tools\teams-tool\
├─ teams_chat_rw.py
├─ teams_requirements.txt
└─ .env                ← TEAMS_CHAT_URL='https://teams.microsoft.com/l/chat/19:...@thread.v2/...'

.env 작성 예

TEAMS_CHAT_URL='https://teams.microsoft.com/l/chat/19:<thread-id>@thread.v2/conversations?context=...'

채팅 URL은 Teams 웹에서 채팅방 우측 ⋯ → 채팅 링크 복사로 얻을 수 있습니다.

⑤ 사용 순서 (PowerShell)

1) 가상환경 만들고 selenium 설치

cd C:\Tools\teams-tool
python -m venv .venv
.\.venv\Scripts\python -m pip install -r teams_requirements.txt

2) 최초 1회 로그인 (브라우저에서 SSO + MFA)

.\.venv\Scripts\python teams_chat_rw.py login --login-wait 180 --timeout 30

Chrome/Edge 창이 뜨면 회사 계정으로 로그인 → MFA 완료 → 창은 180초 후 자동 종료. 로그인 상태는 사용자 프로필 캐시(%USERPROFILE%\.cache\teams-chat-rw\chrome-profile)에 저장됩니다.

3) 최신 메시지 읽기

# .env 의 TEAMS_CHAT_URL 사용
.\.venv\Scripts\python teams_chat_rw.py read --limit 10 --json

# 다른 채팅 한 번만 보고 싶을 때
.\.venv\Scripts\python teams_chat_rw.py read `
  --chat-url 'https://teams.microsoft.com/l/chat/19:<other>@thread.v2/...' `
  --limit 5 --json

4) 메시지 보내기

# 인자로 직접
.\.venv\Scripts\python teams_chat_rw.py write --message "안녕하세요"

# 파일 내용으로
.\.venv\Scripts\python teams_chat_rw.py write --message-file .\payload.txt

# 입력만 하고 전송 보류 (사람이 직접 Send)
.\.venv\Scripts\python teams_chat_rw.py write --message "초안 검토" --no-send

⑥ 주의사항

세션마다 fresh login 권장

토큰이 만료되면 read가 오류 없이 어제·그제 메시지를 “최신”인 양 돌려주는 침묵형 실패가 발생합니다. 작업 시작 전 한 번 더 login 하는 것이 가장 안전합니다.

프로필 폴더 통째로 재생성

로그인 갱신이 안 먹을 때(driver.refresh()로 안 됨)는 캐시 폴더를 지우고 다시 로그인:

Remove-Item -Recurse -Force "$env:USERPROFILE\.cache\teams-chat-rw\chrome-profile"
.\.venv\Scripts\python teams_chat_rw.py login --login-wait 180

회사 정책 / 디바이스 인증서

SSO가 디바이스 인증서·Intune 컴플라이언스를 요구하는 환경에서는 첫 로그인 단계에서 막힐 수 있습니다. 그 경우 동일 계정으로 Edge/Chrome에 일반 로그인이 되는지 먼저 확인하세요.

비밀 파일 관리

.env와 캐시 프로필 폴더는 인증 토큰을 포함합니다. 버전관리에 올리지 말고, 필요 시 BitLocker 등으로 디스크 자체를 암호화해 두세요.

⑦ 트러블슈팅

read가 비어 있거나 어제 메시지만 보임

토큰 만료 또는 “Apply and restart / Accept and login” 배너가 떠 있는 상태. 프로필 폴더 삭제 후 login 재실행.

Chrome/Edge가 자동으로 안 떠요

둘 다 미설치이거나 PATH에 없는 경우. 설치 후 --browser chrome 또는 --browser edge로 명시.

SSL/프록시 오류로 selenium 설치 실패

사내 프록시 환경이면:

.\.venv\Scripts\python -m pip install --proxy http://proxy.company:8080 -r teams_requirements.txt
한글 메시지 인코딩 깨짐 (PowerShell)

PowerShell 콘솔을 UTF-8로:

chcp 65001
$OutputEncoding = [System.Text.Encoding]::UTF8
드라이버 수동 지정이 필요할 때

일반적으로 Selenium Manager가 자동 처리하지만, 사내 망에서 인터넷 다운로드가 막혀 있다면 Chrome/Edge 버전에 맞는 드라이버를 미리 받아 PATH에 두면 됩니다.
