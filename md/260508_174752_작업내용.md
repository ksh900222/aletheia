# 요청사항 요약
- **사용자 질문**: 현재 브랜치가 3개인데 왜 graph에 갈라진게 안보여?

# 작업내용 및 답변

터미널에서 실제 Git 커밋 기록(`git log --graph --all`)을 확인해 본 결과, **세 개의 브랜치가 모두 하나의 일직선(Linear) 상에 놓여있기 때문**입니다.

### 1. Git Graph가 갈라지려면?
그래프에서 나뭇가지처럼 선이 양갈래로 갈라지려면(분기, Diverge), **같은 과거의 커밋에서 출발하여 두 브랜치가 서로 다른 커밋들을 각각 추가해야** 합니다.

### 2. 현재 사용자님의 커밋 상태 (일직선 구조)
현재 사용자님의 커밋 역사를 보면 과거부터 지금까지 일렬로 쭉 쌓여 있습니다. (아래에서 위로 올라갈수록 최신)

```text
* (coworkServer 브랜치 최신) peer 목록 부트스트랩...
* 팀원 관리 페이지 UI...
* 팀 협업 모드...
* (main 브랜치 최신) 스케줄 바 우측 상태 적용
* 간트 차트 내 글자 수...
* 리포트 입력 팝업...
* (design 브랜치 최신) log
* log
...
```

즉, **`design` 브랜치가 있던 자리에서 그대로 이어서 `main` 브랜치의 커밋들이 쌓였고, 다시 그 위로 `coworkServer` 브랜치의 커밋들이 쌓인 상태**입니다. (과거 - 현재 - 미래 처럼요)

결과적으로 세 브랜치가 서로 엇갈린 작업 없이 **순차적으로 같은 줄기로만 성장했기 때문에 그래프가 갈라지지 않고 일직선으로만 보이는 것**입니다.

### 💡 갈라지는 것을 보시려면?
만약 지금 상태에서 `main` 브랜치로 이동(`git checkout main`)하신 다음, 새로운 코드를 작성하고 커밋을 하시게 되면, 그때부터는 `coworkServer`가 가고 있는 길과 `main`이 가는 길이 달라지게 되므로 **그래프가 Y자 모양으로 갈라지는 것**을 확인하실 수 있습니다.
