# 요청사항 요약
- **사용자 질문**: `git config --global core.editor "code --wait"` 명령어를 터미널에 입력해야 하는지 문의. (코딩애플 강의 내용 중 일부)

# 작업내용 및 응답
결론부터 말씀드리면, **절대 입력하시면 안 됩니다! 해당 명령어는 터미널에 입력하지 말고 무시하고 넘어가셔야 합니다.**

### 💡 입력하면 안 되는 이유
- 해당 명령어는 Git이 긴 커밋(commit) 메시지를 작성하거나 충돌을 해결할 때 사용할 **기본 텍스트 에디터를 VS Code로 설정**하겠다는 뜻입니다.
- 앞서 말씀드린 것처럼 현재 사용자님의 PC에는 VS Code 전용 명령어인 `code`가 연결되어 있지 않습니다.
- 만약 이 명령어를 그대로 터미널에 입력해버리면, 나중에 Git이 에디터를 띄워야 하는 상황에서 `code`라는 프로그램을 찾지 못해 **에러를 뿜으며 Git 작업이 중간에 멈추는 불상사가 발생**하게 됩니다.

### 💡 어떻게 해야 할까요?
- **아무것도 입력하지 않고 그냥 다음 강의 내용으로 넘어가시면 됩니다.** 
- 맥(macOS)에 내장된 Git은 기본적으로 시스템에 설치된 터미널 에디터(Vim 또는 Nano)를 사용하도록 알아서 잘 설정되어 있습니다. 따라서 굳이 에디터를 변경하는 설정을 하실 필요가 없습니다.

**[핵심 요약]**
강의를 보실 때 명령어 안에 **`code`**라는 단어가 들어가거나 **VS Code의 메뉴를 클릭하라는 안내**가 나오면, 이는 "VS Code 사용자 전용"이므로 **전부 무시**하시면 됩니다. 
대신 이름과 이메일을 등록하는 `git config --global user.name "내이름"` 과 같은 **순수한 Git 설정 명령어들만** 따라서 터미널에 입력해 주세요!
