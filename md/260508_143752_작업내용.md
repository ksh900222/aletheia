# 요청사항 요약
- **사용자 질문**: git graph라는 것이 있는데, 이것은 commit한 게 보여지는거야? 아니면 push한 것이 보여지는거야

# 작업내용 및 답변

결론부터 말씀드리면, **git graph는 'commit한 내역'을 보여주는 것입니다.** 

더 상세히 설명해 드리자면 다음과 같습니다:

1. **Commit 내역의 시각화**: git graph(또는 터미널의 `git log --graph` 같은 도구)는 현재 내 컴퓨터(로컬)에 기록된 모든 커밋(commit)들의 흐름과 브랜치(branch)가 갈라지고 합쳐지는 과정을 시각적으로 그려줍니다. 따라서 **push를 하지 않더라도 내가 commit만 했다면 그 내역은 즉시 그래프에 나타납니다.**

2. **Push 여부(원격 저장소 상태)도 함께 확인 가능**: 그래프 상에는 각각의 커밋을 가리키는 '라벨(태그)'들이 표시됩니다. 
   - **내 컴퓨터의 브랜치 (예: `main`, `design`)**
   - **원격 저장소(GitHub 등)의 브랜치 (예: `origin/main`, `origin/design`)**
   
   만약 내가 새로 커밋을 여러 개 했다면 내 로컬 브랜치 라벨(`main`)은 최신 커밋 위치로 이동하지만, `origin/main` 라벨은 과거 커밋에 머물러 있게 됩니다. 이를 통해 **"아, 이 커밋들은 아직 push가 안 된 내 로컬 커밋들이구나"**라는 것을 시각적으로 파악할 수 있습니다. 
   이후에 push를 완료하면 `origin/main` 라벨도 최신 커밋 위치로 따라 올라오게 됩니다.

**요약하자면:** 
git graph는 **기본적으로 내 컴퓨터의 commit 내역**을 보여주며, 그 위에 원격 저장소(`origin`)의 위치를 라벨로 함께 표시해주기 때문에 **어떤 커밋이 push되었고 어떤 커밋이 아직 로컬에만 있는지 한눈에 비교**할 수 있게 해주는 아주 유용한 도구입니다.
