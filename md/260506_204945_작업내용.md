# 요청사항 요약
- **사용자 질문**: 로컬 커밋을 성공한 뒤 `git pull`을 실행했으나, `hint: You have divergent branches and need to specify how to reconcile them.` 에러가 발생하여 해결 방법 문의.

# 작업내용 및 응답
우선 터미널 기록을 보니 직전에 안내해 드린 `git commit`을 아주 완벽하게 성공하셨습니다! 정말 잘하셨습니다.

지금 뜬 에러는 오류라기보다는 Git이 던지는 질문입니다. **"현재 내 PC에도 방금 네가 만든 새로운 커밋이 있고, 깃허브 쪽에도 내가 받아올 커밋이 7개나 있어서 나뭇가지처럼 두 갈래로 쪼개졌어(divergent). 이거 두 개를 어떤 방식으로 합쳐줄까?"** 라고 물어보는 정상적인 과정입니다.

### 💡 어떻게 해결할까요?
가장 안전하고 전 세계적으로 많이 쓰는 **"병합(Merge)" 방식**으로 합치겠다고 Git에게 한 번만 설정해 주시면 됩니다. 터미널에 아래 두 줄의 명령어를 차례대로 쳐주세요!

```bash
# 1. 앞으로 두 갈래로 나뉘면 기본적으로 '병합(Merge)' 방식으로 합치겠다고 설정해 줍니다.
git config pull.rebase false

# 2. 설정을 마쳤으니 다시 코드를 땡겨와서 하나로 합칩니다!
# (주의: 뒤에 --no-edit을 꼭 붙여주세요. 안 그러면 알 수 없는 메모장 화면이 떠서 당황하실 수 있습니다.)
git pull --no-edit
```

위 두 줄을 입력하시면, 방금 사용자님이 내 PC에서 수정한 코드와 깃허브의 코드가 안전하게 하나로 합쳐져서 최신 상태가 됩니다. 바로 입력해 보세요!
